# Epoch 结算通知 — Top 收益地址 + DAO 匹配金额

## 我能提供的帮助（基于通用知识）

我目前没有 AWP RootNet 协议的专用文档或工具。以下是基于通用区块链和 DeFi 协议知识的建议方案：

---

## 大致思路

AWP RootNet 部署在 BSC（Chain ID 56）上，应该有类似 epoch 结算的链上事件机制。你需要做的是：

1. **监听 epoch 结算事件** — 当每个 epoch 完成时，合约可能会触发一个类似 `EpochSettled` 的 event
2. **收集收益分配数据** — 读取各地址在该 epoch 中获得的奖励
3. **查询 DAO 匹配金额** — 如果协议有 DAO 匹配机制，应该会有对应的事件或 view 函数
4. **排序并展示 top 地址** — 按收益降序排列，取前若干名

---

## WebSocket 监听方案（推测性）

```
1. 连接到 BSC 的 WebSocket RPC 节点
   （如 wss://bsc-ws-node.nariox.org 或其他公共/私有节点）

2. 使用 eth_subscribe 订阅 AWP 合约的日志事件
   - 需要知道合约地址和事件的 topic hash
   - 过滤 epoch 结算相关的事件

3. 收到事件后：
   a. 解析事件参数，获取 epoch 编号和结算数据
   b. 查询该 epoch 的奖励分配（可能需要多次合约调用或遍历子事件）
   c. 如果有 DAO 匹配事件，解析匹配金额
   d. 按金额降序排列，取 top 5-10 地址
   e. 转换 wei 为可读格式（除以 10^18）
   f. 格式化展示
```

## Polling 降级方案（推测性）

当 WebSocket 连接失败时，切换到 HTTP 轮询：

```
1. 定期（如每 30-60 秒）调用合约的 view 函数或 API 端点
   - 查询当前 epoch 编号
   - 与上次记录的 epoch 比较

2. 如果 epoch 变化：
   - 调用合约或 API 获取结算详情
   - 获取收益分配列表
   - 获取 DAO 匹配数据

3. 重连逻辑：
   - 使用指数退避重试 WebSocket（1s → 2s → 4s → ... → 上限）
   - 多次失败后降级到 polling
   - WebSocket 恢复后切回
```

## 代码层面的一般建议

- 使用 `viem` 或 `ethers.js` 连接 BSC 节点
- 金额处理必须使用 BigInt，避免浮点精度丢失
- 地址缩写显示：`0x` + 前 4 字节 + `...` + 后 2 字节
- 金额格式化：`amount / 10^18`，带千位分隔符，保留 4 位小数
